API Dokumentáció

Az alábbi táblázat az alkalmazás összes publikus és védetett API végpontját dokumentálja. A /api prefixet minden URL automatikusan tartalmazza.

Publikus végpontok (auth nélkül)

Metódus URL Paraméterek Leírás Válasz
GET /rents query: city, county, price_min, price_max, bedrooms, bathrooms, area_min, area_max, available_from Összes elérhető hirdetés listázása (status=available) szűrési lehetőséggel JSON array of rents with owner, images, utilities, reviews
GET /rents/{id} — Egy konkrét hirdetés részletes adatai Rent object + owner details + utilities array + rent_images array + reviews
GET /rent-types — Lakástípusok (apartman, ház, stúdió, garzon) [{id: 1, name: "Apartman"}, ...]
GET /counties — Megyék listája [{id: 1, name: "Budapest"}, ...]
GET /cities — Városok listája [{id: 1, name: "Budapest"}, ...]
GET /utility-options — Felszereltség opciók (WiFi, garázs, balkon, stb.) [{id: 1, name: "WiFi"}, ...]
POST /contact body: name*, email*, phone?, message* Kapcsolatfelvételi form beküldése (szerver-oldali logging) {message: "Üzenet sikeresen elküldve!", status: "success"}
POST /register body: name*, email*, password*, password_confirmation* Új felhasználó regisztrációja {user: {...}, access_token: "..."}
POST /login body: email*, password* Bejelentkezés email/jelszó alapján {user: {...}, access_token: "..."}

Védetett végpontok (Sanctum Bearer token)

Minden végpontnál szükséges: Authorization: Bearer {access_token} header

Metódus URL Paraméterek Leírás Jogosultság
GET /user — Jelenlegi bejelentkezett user adatai Bejelentkezett user
PUT /user body: name?, email?, password?, password_confirmation? Profil módosítása (email nem módosítható) Bejelentkezett user
POST /logout body: {} Kijelentkezés, token revokáció Bejelentkezett user
POST /rents body: title*, description?, price*, currency*, city*, county*, rent_type*, address*, area?, bedrooms*, bathrooms*, available_from*, utility_ids?[] Új hirdetés létrehozása (status auto: available/draft) Bejelentkezett user (owner_id = user.id)
PUT /rents/{id} body: (mint create) Hirdetés módosítása Owner vagy Admin
DELETE /rents/{id} — Hirdetés törlése (cascade: images, rentings, reviews) Owner vagy Admin
GET /user/rents — Bejelentkezett user saját hirdetéseit listázza Bejelentkezett user
POST /upload body: images[]*, rent_id*, default_image_index? (multipart/form-data) Képfeltöltés egy hirdetéshez (Base64 vagy binary) Owner vagy Admin
DELETE /images/{rent_image} — Egy kép törlése Owner vagy Admin
POST /rentings body: rent_id*, user_id* Bérlés kezdeményezése Bejelentkezett user
GET /user/rentings — Bejelentkezett user bérléseit listázza Bejelentkezett user
GET /user/ownerships — Bejelentkezett user tulajdonosi bérléseit listázza Bejelentkezett user
DELETE /rentings/{id} — Bérlés törlése Owner vagy Admin
POST /reviews body: renting_id*, rating* (1-5), comment? Értékelés írása (PK = renting_id) Bejelentkezett user
PUT /reviews/{id} body: rating?, comment? Értékelés módosítása Author
DELETE /reviews/{id} — Értékelés törlése Author vagy Admin

DELETE /admin/rents/{id} — Hirdetés törlése
GET /admin/rentings — Összes bérlés
DELETE /admin/rentings/{id} — Bérlés törlése
GET /admin/reviews — Összes értékelés
DELETE /admin/reviews/{id} — Értékelés törlése
GET /admin/counties — Összes megye
POST /admin/counties body: name* Megye létrehozása
PUT /admin/counties/{id} body: name* Megye módosítása
DELETE /admin/counties/{id} — Megye törlése
GET /admin/cities — Összes város
POST /admin/cities body: name*, county_id* Város létrehozása
PUT /admin/cities/{id} body: name?, county_id? Város módosítása
DELETE /admin/cities/{id} — Város törlése
GET /admin/rent-types — Összes lakástípus
POST /admin/rent-types body: name* Lakástípus létrehozása
PUT /admin/rent-types/{id} body: name* Lakástípus módosítása
DELETE /admin/rent-types/{id} — Lakástípus törlése
GET /admin/utility-options — Összes felszereltség opció
POST /admin/utility-options body: name* Opció létrehozása
PUT /admin/utility-options/{id} body: name* Opció módosítása
DELETE /admin/utility-options/{id} — Opció törlése

HTTP hibakódok és leírások

Hibakód Szcenária Válasz
200 OK Sikeres GET/PUT/PATCH Objektum vagy objektum array
201 Created Sikeres POST (új erőforrás) Új objektum + location header
400 Bad Request Malformed JSON/URL {"message": "The given data was invalid.", "errors": {...}}
401 Unauthorized Missing/invalid token {"message": "Unauthorized"}
403 Forbidden Token valid, de nincs jogosultság {"message": "This action is not authorized."}
404 Not Found Resource nem létezik {"message": "Not found"}
422 Unprocessable Entity Validáció hiba {"message": "The given data was invalid.", "errors": {field: [...]}}
500 Internal Server Error Szerver hiba {"message": "Server error"}
